class DQNetwork(nn.Module):
    def __init__(self, state_size, action_size):
        super(DQNetwork, self).__init__()
        self.fc1 = nn.Linear(state_size, 24)
        self.fc2 = nn.Linear(24, 24)
        self.fc3 = nn.Linear(24, action_size)
        
        
    def forward(self, state):
        x = torch.relu(self.fc1(state))
        x = torch.relu(self.fc2(x))
        return self.fc3(x)

def epsilon_greedy_action(state, epsilon):
    if random.random() <= epsilon:
        return random.choice(range(action_size))
    else:
        with torch.no_grad():
            state = torch.tensor(state, dtype=torch.float32).unsqueeze(0)
            q_values = policy_net(state)
        return torch.argmax(q_values).item()

def replay_experience(batch_size):
    if len(memory) < batch_size:
        return
    batch = random.sample(memory, batch_size)
    
    states, actions, rewards, next_states, dones = zip(*batch)
    
    states = torch.tensor(np.array(states), dtype=torch.float32)
    actions = torch.tensor(actions, dtype=torch.int64).unsqueeze(1)
    rewards = torch.tensor(rewards, dtype=torch.float32)
    next_states = torch.tensor(np.array(next_states), dtype=torch.float32)
    dones = torch.tensor(dones, dtype=torch.float32)

    # Calculamos Q-valores actuales y el objetivo
    current_q_values = policy_net(states).gather(1, actions)
    with torch.no_grad():
        next_q_values = target_net(next_states).max(1)[0]
        target_q_values = rewards + gamma * next_q_values * (1 - dones)
    
    loss = nn.functional.mse_loss(current_q_values.squeeze(), target_q_values)
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Defina en qu consiste y en qu clase de problemas se pueden usar cada uno de los siguientes acercamientos en Deep Reinforcement Learning

1. Proximal Policy Optimization (PPO)

PPO es un algoritmo de Deep Reinforcement Learning perteneciente a la familia de mtodos basados en polticas .Es una mejor de los mtodos de gradiente de poltica y se usa para aprender una poltica que maximiza las recompensa esperada a largo plazo. Funciona de la siguiente manera: PPO busca actualizar la poltica sin realizar cambios abruptos, buscando no desestabilizar el aprendizaje.

Caractersticas:

  • Usa la razn de probabilidad entre la poltica nueva y la antigua para limitar el cambio en la poltica, con una restriccin de confianza.
  • Maximiza una funcin objetivo que combina el gradiente de polticai y la penalizacin de grandes cambios.

Problemas en los que se puede usar:

  • Problemas de control continuo o discreto.
  • Tareas de navegacin autnoma.
  • Juegos de Atari.
  • Entornos donde hay alta dimensionalidad en los estados y acciones.
  • Entornos donde hay alta incertidumbre.

2. Deep Deterministic Policy Gradient (DDPG)

Es un algoritmo de Deep Reinforcement Learning basado en polticas que extiende el menfoque de Deterministic POlicy Gradient usando redes neuronales profundas. Es un mtodo actor-critic que aprende una poltica determinista.

Caractersticas:

  • El actor aprende una poltica determinstica que selecciona acciones directamente a partir de los estados, sin muestreo.
  • El crtico estima el valor Q de la poltica determinstica aprendida por el actor.
  • Usa tcnicas de estabilizacin como experiencia replay y target networks.

Problemas en los que se puede usar:

  • Problemas de control continuo.
  • Tareas de navegacin autnoma.
  • Manipulacin de objetos.

3. Trust Region Policy Optimization (TRPO)

Es un algoritmo de DRL basado en polticas. Est diseado para garantizar actualizaciones de polticas mas estables y eficientes en el aprendizaje por una poltica ptima. TRPO ajusta la poltica gradualmente para evitar grandes cambios que puedan reducir el rendimiento, de manera que se asegure que cada actualizacin est dentro de una regin de confianza previamente definida.

Caractersticas:

  • Restringe la magnitud de los cambios en la poltica usando una mtrica llamada divergencia de Kullback-Leibler (KL), que mide cun diferente es la nueva poltica respecto a la anterior.
  • Optimiza la poltica directamente, lo que lo hace adecuado para entornos de espacio continuo y discretos.
  • Es ms estable que los mtodos de gradiente de poltica tradicionales, pero computacionalmente ms costoso.

Problemas en los que se utiliza:

  • Robtica y sistemas fsicos donde los cambios abruptos pueden ser perjudiciales.
  • Videojuegos complejos (como juegos de Atari) y simulaciones donde se necesita alta estabilidad en las actualizaciones de polticas.
  • Tareas de control continuo como navegacin, control de drones o vehculos autnomos.

4. Asynchronous Advantage Actor-Critic (A3C)

Es un algoritmo de Deep Reinforcement Learning que combina el enfoque actor-crtico con el aprendizaje asincrnico. A3C aprovecha mltiples agentes o hilos que interactan con el entorno de manera independiente, aprendiendo en paralelo y actualizando la poltica global de forma asincrnica.

Caractersticas:

  • Utiliza un actor que aprende la poltica (seleccin de acciones) y un crtico que estima el valor esperado (ventaja) de las acciones.
  • Los hilos ejecutan copias independientes del entorno, actualizando una poltica global compartida.
  • Calcula la ventaja (Advantage) para medir cun buena fue una accin en comparacin con lo esperado, mejorando la eficiencia del aprendizaje.
  • Asincrona permite una exploracin ms diversa, evitando quedarse atrapado en subptimos locales.

Problemas en los que se utiliza:

  • Problemas de aprendizaje en paralelo y alta dimensionalidad, como en juegos complejos (e.g., Atari, StarCraft).
  • Tareas de control continuo y ambientes fsicos donde se necesita eficiencia computacional (robots, simulaciones fsicas).
  • Es adecuado para problemas que requieren alta exploracin y eficiencia, como entornos de simulacin masiva o multi-agente.
